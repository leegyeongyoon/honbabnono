# 잇테이블 (eattable.kr)

### 선주문 / 선결제형 외식 예약 플랫폼

> **"자리가 아니라, 식사 시작을 예약하다"**

---

## 1. 서비스 소개

| 항목 | 내용 |
|------|------|
| **서비스명** | 잇테이블 (EatTable) |
| **도메인** | eattable.kr |
| **한 줄 정의** | 고객이 매장 방문 전에 좌석, 시간, 인원, 메뉴, 결제를 모두 확정하고, 매장은 고객의 도착 예정 상태를 보며 조리와 상차림 타이밍을 조절해 **"예약한 시간에 빠르게 식사를 시작"**하게 만드는 외식 플랫폼 |
| **슬로건** | "자리가 아니라, 식사 시작을 예약하다" |

### 핵심 차별점

캐치테이블이 **자리**를 예약한다면, 잇테이블은 **메뉴와 식사 시작**까지 예약합니다.

기존 예약 앱에서는 착석 후에도 메뉴 선택, 주문, 조리 대기, 결제라는 과정이 남아 있습니다. 잇테이블은 이 모든 단계를 방문 전에 완료해, 고객은 도착 즉시 차려진 음식과 함께 식사를 시작합니다.

### 슬로건 후보

- 예약한 시간에, 바로 먹는 외식 예약
- 자리가 아니라 식사 시작을 예약하다
- 메뉴까지 예약하는 외식 플랫폼
- 도착 전에 주문하고, 도착하면 바로 식사
- 매장 안 키오스크를 방문 전 스마트폰으로

## 2. 왜 이 아이템이 필요한가

### 기존 예약 앱의 한계

기존 예약 앱이 해결하는 범위:

> 몇 시에 몇 명이 갈지 예약 → 웨이팅 등록 → 노쇼 방지용 예약금 → 방문 확인

하지만 실제 고객 경험은 여기서 끝나지 않습니다:

```
[기존 플로우 — 예약 후에도 남은 단계]
예약 시간 도착 → 착석 → 메뉴판 확인 → 일행과 메뉴 고민 → 직원 호출
→ 주문 접수 → 주방 전달 → 조리 시작 → 음식 제공 → 결제

[잇테이블 플로우]
메뉴 선택 + 결제 완료 → 매장 도착 → QR 체크인 → 즉시 식사 시작
```

**핵심:** 기존 예약은 입장 시간을 줄여주지만, 식사 시작 시간까지 줄여주지는 못합니다. 잇테이블은 **"들어가는 것"이 아니라 "먹기 시작하는 것"에 가까운 경험**을 제공합니다.

### 고객이 겪는 문제

| 문제 | 상세 |
|------|------|
| **예약했는데 여전히 기다림** | 좌석만 확보될 뿐, 착석 후 메뉴 선택-주문-조리까지 15~30분 추가 소요 |
| **일행과 메뉴 합의 지연** | 착석 후 메뉴 고르는 시간이 의외로 길고, 특히 단체 모임에서 심각 |
| **결제 과정의 불편** | 식사 후 계산대 대기, 더치페이 정산 번거로움 |
| **시간 압박** | 점심시간 1시간 내에 모든 것을 처리해야 하는 직장인에게 치명적 |

### 점주가 겪는 문제

| 문제 | 상세 |
|------|------|
| **노쇼(No-show)** | 예약 부도로 인한 재료 폐기, 매출 손실. 평균 노쇼율 약 15~20% |
| **주문 응대 과부하** | 피크타임 주문 집중으로 서빙 인력 부족, 고객 불만 증가 |
| **재료 예측 어려움** | 당일 수요를 사전에 알 수 없어 과다/과소 준비 반복 |
| **회전율 저하** | 착석 후 메뉴 결정-조리-결제까지 테이블 점유 시간이 길어짐 |

### 문제-해결 매핑

| 현재 문제 | 고객 불만 | 점주 불만 | 잇테이블 해결 방식 |
|----------|----------|----------|------------------|
| 예약 후에도 주문 대기 | 예약했는데 또 기다림 | 직원 호출/응대 반복 | 메뉴 사전 선택 |
| 음식 제공 지연 | 일정이 밀림 | 주문 몰림, 주방 혼잡 | 도착 상태 기반 조리 준비 |
| 노쇼 | 동행/예약 깨짐 | 좌석 손실, 재료 손실 | 선결제/보증금/노쇼 정책 |
| 지각 | 음식 식음, 제공 지연 | 조리 타이밍 판단 어려움 | 10분 전/5분 전 도착 알림 |
| 메뉴 결정 지연 | 일행과 현장 고민 | 테이블 회전 저하 | 예약 시 메뉴 확정 |
| 결제/정산 번거로움 | 더치페이 스트레스 | 계산대 혼잡 | 사전 결제/분할결제 |
| 매장 운영 예측 어려움 | 품질 편차 | 재료/인력/좌석 예측 어려움 | 예약 주문판/조리 상태 관리 |

### 시장이 이미 보여주는 흐름

| 영역 | 기존 서비스 | 해결하는 문제 |
|------|------------|-------------|
| 예약/웨이팅 | 캐치테이블, 테이블링 | 입장 전 대기, 좌석 확보 |
| 매장 내 주문 | 티오더, 메뉴잇, 키오스크 | 착석 후 주문/결제 효율화 |
| **잇테이블** | **선주문형 외식 예약** | **예약 단계에서 메뉴/결제/조리 준비까지 연결** |

> 소비자는 이미 "외식 전 과정을 앱으로 미리 처리하는 것"에 익숙해지고 있습니다.
> 잇테이블은 키오스크를 부정하는 게 아니라, **매장 안 키오스크의 주문/결제 기능을 고객 스마트폰과 예약 단계로 앞당기는 서비스**입니다.

## 4. 경쟁사 비교

### vs 캐치테이블

| 구분 | 캐치테이블 | 잇테이블 |
|------|-----------|---------|
| 핵심 가치 | 자리 예약, 웨이팅 관리 | 자리 + 메뉴 + 결제 + 조리 준비 |
| 고객 약속 | 예약 시간에 입장 | 예약 시간에 빠르게 식사 시작 |
| 메뉴 확정 | 보통 방문 후 결정 | 예약 시점에 메뉴 확정 |
| 결제 | 예약금/노쇼 방지 중심 | 메뉴 선결제 중심 |
| 매장 준비 | 좌석 중심 | 좌석 + 메뉴 + 재료 + 조리 타이밍 |
| 노쇼 대응 | 예약금/방문 확인 | 선결제 + 지각/도착 확인 + 조리 상태 기반 환불 |
| 확장성 | 맛집 예약 플랫폼 | 외식 주문 예약 운영 플랫폼 |

> **캐치테이블이 "자리를 예약하는 서비스"라면, 잇테이블은 "먹을 메뉴와 식사 시작을 예약하는 서비스"입니다.**

### vs 테이블오더

| 구분 | 테이블오더 | 잇테이블 |
|------|-----------|---------|
| 주문 시점 | 착석 후 | 방문 전/예약 시 |
| 고객 위치 | 매장 안 | 매장 밖 |
| 목적 | 직원 호출/주문 대기 감소 | 도착 후 주문/조리 대기 감소 |
| 결제 | 식사 중/후 또는 선불 | 예약 단계 선결제 |
| 매장 준비 | 주문 후 시작 | 도착 전 준비 가능 |
| 핵심 가치 | 매장 내 주문 효율화 | 방문 전 주문 예약화 |

> **테이블오더가 매장 안 주문을 디지털화했다면, 잇테이블은 그 주문을 예약 전 단계로 앞당깁니다.**

## 5. 타겟 사용자

### 고객 (B2C, 우선순위)

| 순위 | 세그먼트 | 핵심 니즈 | 적합 기능 |
|------|---------|----------|----------|
| **1순위** | 직장인 점심/저녁 | 1시간 내 식사 완료, 대기 제거 | 시간대별 예약, 세트 메뉴 선결제, 도착 10분 전 알림 |
| **2순위** | 데이트/약속 | 완벽한 식사 경험, 대기 없는 코스 | 코스/세트 선결제, 도착 시간 기반 준비, 지각 알림 |
| **3순위** | 소규모 회식/모임 | 메뉴 사전 합의, 간편 정산 | 대표 예약, 인원별 메뉴 선택, 더치페이/분할결제 |
| **4순위** | 혼자 먹기 어려운 메뉴 고객 | 동행 매칭 + 메뉴/결제 사전 확정 | 메뉴 기반 모임 생성, 참여자별 결제, 보증금 |

#### 직장인 점심 상세

```
점심시간 1시간 = 이동 10분 + 웨이팅 10분 + 주문 5분 + 조리 대기 15분 = 실제 식사 시간 부족
→ "가게 도착하면 바로 먹고 싶다", "팀원들과 메뉴 합의+결제 미리 끝내고 싶다"
```

#### 데이트/약속 고객

```
예약했는데 음식 나오기까지 오래 걸림 → 다음 일정에 영향
→ "예약 시간에 맞춰 식사를 시작하고 싶다", "중요한 날에 실패하고 싶지 않다"
```

#### 회식/단체 모임

```
인원 변동 + 메뉴 결정 지연 + 정산 복잡 + 노쇼/지각
→ "메뉴 미리 정하고 각자 결제", "늦는 사람 때문에 전체가 피해보지 않으면 좋겠다"
```

#### 혼자 먹기 어려운 메뉴 (기존 밥동무 연결)

```
고기, 전골, 샤브샤브, 회, 찜 = 1인 주문 어려움
→ "같이 먹을 사람도 구하고, 메뉴와 결제까지 미리 끝내고 싶다"
```

### 매장 (B2B, 초기 타겟)

| 업종 | 적합도 | 이유 |
|------|--------|------|
| **샤브샤브** | 매우 높음 | 인원/세트/상차림 준비가 명확 |
| **전골/찜** | 높음 | 조리 준비 시간 있고 단체 수요 |
| **고깃집 세트** | 높음 | 기본찬/불판/고기 세팅 가능 |
| **훠궈** | 높음 | 세트화 쉽고 준비 단계 분리 가능 |
| **보쌈/족발/찜닭** | 중간~높음 | 메뉴 확정과 선결제 적합 |
| **코스요리** | 높음 | 선예약/선결제 문화와 맞음 |
| **파스타/스테이크** | 중간 | 체크인 후 조리 필요 |
| **국밥/분식** | 낮음 | 회전 빠르고 사전 예약 가치 낮음 |

> 초기에는 **고기, 샤브샤브, 전골, 찜, 회, 훠궈**처럼 혼자 먹기 어렵고 2인 이상 주문이 자연스러운 메뉴에 집중합니다.

## 6. 핵심 서비스 컨셉

### 서비스 정의

본 서비스는 외식 예약 단계에서 방문 시간, 인원, 메뉴, 결제를 함께 확정하고, 고객의 도착 예정 상태를 매장에 전달해 조리/상차림 타이밍을 조절하는 **선주문/선결제형 외식 예약 플랫폼**입니다.

### 대상별 제공 가치

| 대상 | 제공 가치 |
|------|----------|
| 고객 | 도착 후 메뉴 고민, 주문, 결제 대기 감소 |
| 점주 | 노쇼 감소, 주문 응대 감소, 재료/조리 예측 가능 |
| 주방 | 예약 주문 기반으로 조리 순서 관리 가능 |
| 모임 고객 | 정산 스트레스 감소, 메뉴 사전 합의 가능 |
| 기존 밥동무 고객 | 같이 먹을 사람 모집 후 메뉴까지 확정 가능 |

## 7. 사용자 여정

### 고객 플로우

```
[1단계: 탐색]          [2단계: 예약]           [3단계: 확정]
매장 검색/추천     →   날짜/시간/인원 선택  →   메뉴 선택 + 결제
                                              (카드/간편결제)

[4단계: 이동]          [5단계: 도착]           [6단계: 식사]
실시간 도착 알림   →   QR 체크인           →   즉시 식사 시작
(매장에 ETA 전달)      (매장: 조리 완료)       (대기 시간 0분)
```

| 단계 | 고객 행동 | 매장 행동 | 시스템 동작 |
|------|----------|----------|------------|
| 매장 선택 | 위치/카테고리/추천으로 매장 탐색 | - | 개인화 추천, 실시간 예약 가능 여부 표시 |
| 메뉴 선택 | 메뉴 구성, 수량, 옵션 확정 | - | 재고/가용 시간대 실시간 검증 |
| 결제 | 카드/간편결제로 선결제 | 예약 접수 알림 수신 | 결제 완료, 예약 확정 |
| 이동 중 | 앱에서 "출발" 버튼 탭 | 도착 예정 시간(ETA) 확인 | GPS 기반 ETA 계산, 매장에 전달 |
| 도착 | QR 체크인 | 조리 완료, 상차림 | 체크인 확인, 매장에 알림 |
| 식사 | 즉시 식사 시작 | 서빙 | 테이블 타이머 시작 |
| 완료 | 리뷰 작성 (선택) | 정산 확인 | 매장 정산 처리 |

### 점주 플로우

```
[예약 접수]         [도착 확인]         [조리 시작]         [제공]           [정산]
주문 내역 확인  →   ETA 모니터링    →   도착 10분 전     →  체크인 확인  →   자동 정산

```

### 관리자 플로우

```
매장 승인 → 메뉴 검수 → 예약 모니터링 → 결제/환불 처리 → 노쇼/분쟁 처리 → 매장 정산 → KPI 분석
```

### 결제 금액 구성

```
총 결제금액 = 메뉴 합계 + 옵션 추가금 - 할인(쿠폰/프로모션) - 포인트 사용
```

- 최소 결제 금액: **1,000원**
- 최대 결제 금액: **5,000,000원** (1회)
- 부가세: 메뉴 가격에 **포함** (매장 등록 시 세금 포함가 입력)
- 결제 미완료 예약: **15분 내 미결제 시 자동 취소**

### 일반 예약형 취소/환불

일행이 이미 있는 고객. **메뉴 전액 선결제**, 별도 보증금 없음.

| 취소 시점 | 환불 비율 | 위약금 | 비고 |
|----------|----------|--------|------|
| 예약 **24시간 전** 이전 | **100%** 전액 환불 | 0% | |
| 예약 **3시간~24시간 전** | **90%** 환불 | 10% | 매장 재료 준비 보상 |
| 예약 **1시간~3시간 전** | **70%** 환불 | 30% | |
| 예약 **30분~1시간 전** | **50%** 환불 | 50% | |

**환불 처리 기한:**
- 카드: 취소 접수 후 영업일 3~5일 (카드사별 상이)

- 포인트: 즉시 복원

## 10. 지각/노쇼 관리 정책

### 핵심 원칙

> 선결제는 음식에 대한 권리이고, 예약 시간 준수는 좌석과 빠른 제공에 대한 권리다.
> 무단 지각 시 음식 제공 권리는 남을 수 있지만, 즉시 제공과 좌석 우선권은 사라질 수 있다.

### 상태 정의

| 상태 | 기준 | 처리 |
|------|------|------|
| 정상 | 예약 시간 ±5분 도착 | 정상 제공 |
| 지연 알림 | 고객이 사전 지연 입력 | 조리 보류/조정 |
| 무단 지각 | 알림 없이 +10분 초과 | 즉시 제공 보장 제외 |
| 좌석 우선권 해제 | 알림 없이 +15분 초과 | 매장 재량으로 대기 전환 |
| 노쇼 판정 시작 | 알림 없이 +20분, 연락 불가 | 노쇼 처리 가능 |
| 노쇼 확정 | +30분 (어떤 상태든) | **무조건 노쇼 확정** |
| 체크인 완료 | QR/직원 확인 | 조리 시작/제공 진행 |

### 노쇼 페널티 (고객)

| 누적 횟수 (6개월 기준) | 페널티 |
|----------------------|--------|
| **1회** | 경고 알림 + 신뢰도 하락 |
| **2회** | 예약 시 **추가 보증금 10%** 부과 |
| **3회** | **7일간 예약 제한** |
| **5회** (12개월 기준) | **30일간 예약 제한** + 서비스 경고 |
| **7회** (12개월 기준) | **계정 정지 검토** (관리자 수동 판단) |

**페널티 해제 조건:**
- 6개월간 노쇼 0건 → 누적 카운트 1단계 감소
- 정상 체크인 10회 연속 → 누적 카운트 초기화
- 불가항력 사유 (천재지변, 의료 긴급) → 관리자 접수 후 노쇼 취소 가능

### 매장 취소 누적 제재

| 조건 | 처리 |
|------|------|
| 매장 귀책 취소 **3회/월** | 경고 알림 |
| 매장 귀책 취소 **5회/월** | 임시 비활성화 (7일) |
| 별점 **2.0 미만** (리뷰 20건+) | 개선 요청 → 미개선 시 퇴점 검토 |

### 사용자 안내 문구

> 예약 시간에 맞춰 빠르게 식사를 시작할 수 있도록 매장이 사전 준비합니다.
> 사전 알림 없이 지각할 경우 좌석 배정과 제공 시간이 조정될 수 있으며, 15분 이상 무단 지각 시 예약 우선권이 해제될 수 있습니다.
> 메뉴 결제금은 매장의 조리 시작 여부에 따라 환불 범위가 달라질 수 있습니다.
> 예약 시간으로부터 30분 이상 무단 미도착 시 노쇼로 확정되며, 누적 시 예약 제한이 적용됩니다.

## 13. MVP 범위

### 목표

> **"정말 사람들이 메뉴까지 선결제하고 예약하는가?"를 검증**

### MVP 구성

- 제휴 매장 **3~5곳** (샤브샤브/고깃집 중심)
- 세트 메뉴 **2~3개**/매장
- 고객: 예약 → 메뉴 선택 → 결제 → 도착 알림 → QR 체크인
- 점주: 예약 확인 → 조리 상태 변경 → 도착 확인

### MVP 필수 기능

| 영역 | 기능 |
|------|------|
| 고객 | 매장 선택, 시간 선택, 메뉴 선택, 결제, 도착 알림, QR 체크인 |
| 점주 | 예약 주문 확인, 준비 시작, 조리 시작, 도착 확인, 노쇼 처리 |
| 관리자 | 매장/메뉴 관리, 예약 조회, 환불, 정산 |
| 알림 | 카카오 알림톡, 앱 푸시, 문자 |
| 결제 | 카드/간편결제, 부분환불, 정산 데이터 |

### MVP에서 제외

| 기능 | 제외 이유 |
|------|----------|
| POS 자동 연동 | 초기 검증에는 과함 |
| GPS 자동 추적 | 개인정보 부담, 수동 버튼으로 대체 가능 |
| AI 조리 예측 | 데이터 쌓인 뒤 가능 |
| 모든 메뉴 등록 | 운영 복잡도 증가 |
| 전국 확장 | 초기에는 상권 집중 필요 |
| 복잡한 좌석 최적화 | 시간 슬롯 기반으로 시작 가능 |

### 앱만으로 가능한가?

**초기에는 앱만으로 가능합니다.**

- 고객 앱에서 예약/메뉴/결제
- 점주용 웹에서 예약 주문 확인
- 점주가 기존 POS에 수동 입력
- 고객 도착 시 QR 체크인
- 점주가 준비/조리/제공 상태 변경
- 관리자 어드민에서 환불/정산 처리

장기적으로 POS/주방프린터 연동이 중요하지만, 초기에는 **점주용 예약 주문판**이 있으면 충분합니다.

## 19. 사업계획서용 핵심 문구

### 문제 정의

> 기존 외식 예약 서비스는 고객이 원하는 시간에 좌석을 확보하는 데 집중해 왔다. 그러나 실제 외식 경험에서는 착석 이후에도 메뉴 선택, 직원 호출, 주문 접수, 조리 시작, 결제 과정에서 추가 대기 시간이 발생한다. 국내에서는 식당 예약/웨이팅 앱 사용자가 최근 3년간 185% 증가했고, 캐치테이블 웨이팅은 누적 웨이팅 1,710만 건 이상을 기록하는 등 외식 대기 문제를 해결하려는 수요가 빠르게 증가하고 있다. 또한 외식업체는 고객 대기시간 감소를 목적으로 키오스크와 테이블오더를 도입하고 있으며, 스마트오더 도입 사례에서는 고객 응대시간 감소 효과도 확인된다. 본 서비스는 이러한 흐름을 매장 착석 이후가 아닌 예약 단계로 앞당겨, 좌석 예약과 메뉴 선결제, 도착 확인, 조리 준비, 노쇼 관리를 통합 제공한다.

### 솔루션 정의

> 본 서비스는 사용자가 외식 예약 단계에서 방문 시간, 인원, 메뉴를 선택하고 결제까지 완료할 수 있는 선주문형 외식 예약 플랫폼이다. 매장은 예약 정보를 기반으로 좌석 배정, 재료 준비, 상차림, 조리 타이밍을 사전에 관리할 수 있으며, 고객의 도착 예정 상태를 확인해 음식 품질 저하와 노쇼 리스크를 줄일 수 있다. 이를 통해 고객은 매장 도착 후 주문/결제 대기 없이 빠르게 식사를 시작하고, 매장은 주문 응대 부담, 좌석 손실, 재료 낭비, 정산 부담을 줄일 수 있다.

### 최종 한 문장

> 이 서비스는 고객이 외식 전에 시간/좌석/메뉴/결제를 확정하고, 매장은 고객의 도착 상태에 맞춰 조리 준비를 조절함으로써, 예약 후에도 발생하는 주문/결제/조리 대기와 노쇼 리스크를 동시에 줄이는 선주문형 외식 예약 플랫폼이다.
