# 혼밥시러 핵심 차별화 기능 정리

> 사업기획서 작성용 - 2024-01-23

---

## Executive Summary

**혼밥시러**는 단순한 밥모임 매칭을 넘어, **신뢰 기반 소셜 다이닝 플랫폼**으로서 다음과 같은 핵심 차별화 요소를 갖추고 있습니다:

| 핵심 가치 | 설명 |
|-----------|------|
| **신뢰성** | 약속금 시스템 + 다중 출석 인증으로 노쇼 최소화 |
| **공정성** | 노쇼 배상금 분배로 피해자 보호 |
| **게이미피케이션** | 밥알지수 + 뱃지로 건전한 참여 유도 |
| **편의성** | AI 검색 + 맞춤 추천으로 최적의 모임 매칭 |

## 1. 약속금 시스템 (Promise Deposit) - 핵심 차별화

### 1.1 개념
모임 참가 시 소액의 **보증금을 예치**하여 실제 참석을 담보하는 시스템

### 1.2 차별화 포인트

```
기존 서비스: 무료 참가 신청 → 노쇼 빈발 → 신뢰 붕괴
혼밥시러:    약속금 예치 → 책임감 상승 → 노쇼 감소 → 신뢰 문화 형성
```

### 1.3 약속금 구조

| 금액 | 대상 모임 | 특징 |
|------|----------|------|
| **0원** | 캐주얼 모임 | 진입 장벽 낮춤 |
| **3,000원** | 소규모 모임 | 부담 없는 책임감 |
| **5,000원** | 일반 모임 (기본 권장) | 적정 수준의 보증 |
| **10,000원** | 프리미엄 모임 | 높은 신뢰도 요구 |

### 1.4 핵심 원칙: 호스트도 동일 금액 납부
- **공정성 확보**: 호스트와 참가자 모두 동등한 책임
- **호스트 노쇼 방지**: 호스트 이탈 시 더 강력한 페널티

### 1.5 시간대별 환불 정책 (투명한 정책)

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
|        100% 환불          | 60%  | 30%  |  0%   | 불가 |
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                           -60분  -40분  -20분  -10분  모임시작
```

### 1.6 비즈니스 모델 연계

| 상황 | 처리 | 수익 |
|------|------|------|
| 정상 완료 | 100% 환불 | - |
| 직전 취소 | 부분 몰수 | 몰수금의 30% |
| **노쇼** | **전액 몰수** | **몰수금의 30%** |

## 2. 노쇼 배상 시스템 - 업계 최초

### 2.1 핵심 컨셉
> "노쇼의 피해는 피해자에게 돌아간다"

### 2.2 배상금 분배 공식

```javascript
노쇼자 약속금 = 피해자 배상(70%) + 플랫폼 수수료(30%)
```

### 2.3 실제 예시

```
[시나리오] 5,000원 약속금 모임, 4명 중 1명 노쇼

노쇼자 몰수금: 5,000원
├─ 피해자 배상 총액: 3,500원 (70%)
│  └─ 3명에게 각 1,166원 포인트 지급
└─ 플랫폼 수익: 1,500원 (30%)
```

### 2.4 차별화 효과
- **피해자 심리적 보상**: 단순 불쾌감 → 실제 보상
- **노쇼자 강력한 억지력**: 금전적 손실 + 신뢰도 하락
- **플랫폼 지속가능성**: 운영 비용 충당

## 3. 다중 출석 인증 시스템 - 기술적 차별화

### 3.1 5가지 체크인 방식

| 방식 | 기술 | 신뢰도 | 상황 |
|------|------|--------|------|
| **GPS 체크인** | Haversine 거리 계산 | ★★★★★ | 야외/GPS 양호 |
| **QR 코드** | 10분 유효 동적 코드 | ★★★★★ | 실내/지하 |
| **호스트 확인** | 수동 승인 | ★★★★☆ | 기기 문제 시 |
| **상호 확인** | 참가자 간 교차 확인 | ★★★★☆ | 호스트 부재 |
| **자가 보고** | 본인 응답 | ★★☆☆☆ | 최후 수단 |

### 3.2 GPS 체크인 기술 상세

```javascript
// Haversine 공식으로 정밀 거리 계산
거리 제한: 100m 이내
정확도: 지구 곡률 반영 (반경 6,371km 기준)
```

### 3.3 QR 코드 보안

```javascript
{
  meetupId: "uuid",
  hostId: "uuid",
  timestamp: 1706000000000,
  expiresAt: 1706000600000  // 10분 후 자동 만료
}
// Base64 인코딩 → 변조 방지
```

### 3.4 노쇼 판정 2단계 검증

```
┌────────────────────────────────────┐
│     1차: GPS/QR 인증 기록 확인      │
│     └─ 미인증 → 노쇼 의심           │
└────────────────────────────────────┘
                 ↓
┌────────────────────────────────────┐
│     2차: 참가자 후기 교차 검증       │
│     ├─ 호스트 신고 → 즉시 확정       │
│     └─ 2명 이상 신고 → 확정         │
└────────────────────────────────────┘
```

## 4. 밥알지수 (Rice Index) - 신뢰도 시스템

### 4.1 개념
사용자의 **활동 이력과 신뢰도를 0-100점**으로 표현하는 독자적 신뢰 지표

### 4.2 레벨 체계 (브랜딩 요소)

| 점수 | 레벨명 | 이모지 | 의미 |
|------|--------|--------|------|
| 98.1+ | **밥神 (밥신)** | 🍚×7 | 전설적인 유저 |
| 90.0-98.0 | **찰밥대장** | 🍚×6 | 거의 완벽한 이력 |
| 80.0-89.9 | **밥도둑 밥상** | 🍚×5 | 최고의 매너 |
| 70.0-79.9 | **고봉밥** | 🍚×4 | 꾸준한 출석 |
| 60.0-69.9 | **따끈한 밥그릇** | 🍚×3 | 양호한 활동 |
| 40.0-59.9 | **밥 한 숟갈** | 🍚×1 | 평균 유저 |
| 0-39.9 | **티스푼** | 🍚×2 | 신뢰 낮음 |

### 4.3 점수 산정 요소

| 요소 | 영향 | 설명 |
|------|------|------|
| 모임 참가 | +점수 | 참여 횟수에 비례 |
| 모임 주최 | +점수 | 호스팅 경험 |
| 리뷰 작성 | +점수 | 커뮤니티 기여 |
| 받은 평점 | +/-점수 | 타인 평가 반영 |
| **노쇼** | **-15점** | 강력한 페널티 |

### 4.4 활용

- 프로필에 레벨 뱃지 표시
- 프리미엄 모임 참가 조건
- 호스트 신뢰도 판단 지표

## 5. 뱃지 시스템 - 게이미피케이션

### 5.1 카테고리별 뱃지

#### 모임 참여 뱃지
| 뱃지명 | 조건 | 아이콘 |
|--------|------|--------|
| 첫 만남 | 1회 | 🍚 |
| 밥친구 | 5회 | 🍜 |
| 단골손님 | 10회 | 🍱 |
| 소셜킹 | 30회 | 👑 |
| 밥신 | 100회 | 🏆 |

#### 리뷰 작성 뱃지
| 뱃지명 | 조건 |
|--------|------|
| 첫 리뷰 | 1회 |
| 후기왕 | 10회 |
| 리뷰 마스터 | 50회 |

### 5.2 진행률 시각화

```
밥친구 뱃지 [████████░░░░░░░░░░░░] 40% (2/5)
```

### 5.3 대표 뱃지 설정
- 획득한 뱃지 중 1개를 **프로필 대표 뱃지**로 설정
- 다른 사용자에게 자신의 성과 표시

### 5.4 확장 계획
- 연속 출석 뱃지 (7일, 30일)
- 카테고리 마스터 (한식 마스터, 양식 마스터)
- 시즌/이벤트 한정 뱃지

## 6. AI 기반 스마트 기능

### 6.1 자연어 AI 검색

```
사용자 입력: "강남에서 점심에 삼겹살 먹을 사람"
     ↓
AI 의도 파악 (GPT-3.5):
{
  category: "한식",
  location: "강남",
  time: "점심",
  keywords: ["삼겹살"]
}
     ↓
최적 모임 추천
```

### 6.2 AI 챗봇 어시스턴트

```
[혼밥시러 AI] 안녕하세요! 무엇을 도와드릴까요?

- 모임 검색 도움
- 앱 사용법 안내
- 맛집 추천
- 참여 관련 질문 답변
```

### 6.3 개인화 모임 추천

```javascript
추천 알고리즘:
1. 사용자 참여 이력 분석
2. 자주 참여한 카테고리 상위 3개 추출
3. 유사 카테고리 모임 우선 추천
4. 이미 참여한 모임 제외
```

### 6.4 카카오 장소 API 연동

- 키워드 검색 (장소명, 업체명)
- 주소 검색 (도로명, 지번)
- 중복 제거 및 정규화

| 취소 시점 | 페널티 |
