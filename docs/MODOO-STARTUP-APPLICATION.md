## Q1. 나의 아이디어를 한 줄로 소개해주세요. (필수, 최소 10자)

**매장 방문 전 메뉴 선택·선결제를 완료하고 도착 즉시 식사를 시작하는 선주문형 외식 예약 플랫폼**

> **아이템명 체크:** "선주문형 외식 예약 플랫폼"
> - 기술: 선주문/선결제 + 도착 기반 조리 타이밍 관리
> - 대상: 외식 고객(직장인/데이트/모임) + 외식 매장(샤브샤브/고깃집/전골)
> - 수준: MVP 개발 완료, 제휴 매장 입점 준비 단계

## Q2. 아이디어를 떠올린 배경 이야기를 들려주세요. (필수, 1000자)

### [개인의 경험에서 출발한 문제 인식]

점심시간 1시간. 동료 4명과 회사 근처 샤브샤브집을 예약하고 갔습니다. 예약했으니 바로 먹을 줄 알았는데, 착석 후 메뉴판 돌려보기 5분, 메뉴 합의 10분, 직원 호출 대기 5분, 주문 후 조리 대기 15분. 결국 음식이 나온 건 도착 35분 후였고, 10분 만에 허겁지겁 먹고 돌아왔습니다. "예약이 도대체 뭘 예약한 거지?"라는 의문이 이 아이디어의 시작이었습니다.

### [가설 수립 → 시장 검증: 이건 나만의 문제가 아니다]

이 불편함이 개인적 경험인지, 시장이 원하는 것인지 검증했습니다.

**[사회적 변화 - S]** 식당 예약·웨이팅 앱 이용자가 3년 만에 102만→291만 명으로 **185% 폭증**했습니다(와이즈앱·리테일, 2025.08). 캐치테이블은 MAU 300만 명을 돌파했고, 누적 웨이팅 1,710만 건을 기록했습니다. 소비자들은 이미 "외식의 불편함을 앱으로 해결하는 것"에 익숙해졌습니다. 하지만 이 모든 서비스가 해결하는 건 **"자리"**뿐입니다. 착석 후 메뉴 결정·주문·조리 대기 15~30분은 여전히 남아있습니다.

**[기술적 변화 - T]** 매장 내 디지털 주문은 이미 표준이 되었습니다. 테이블오더 1위 '티오더'는 누적 결제액 **10조 원을 돌파**했고, 태블릿 26만 대가 전국 매장에 설치되어 있습니다(전자신문, 2025.04). 스타벅스 사이렌 오더는 누적 **7억 건**을 넘기며 고객 10명 중 4명이 사용합니다. "미리 주문하고 매장에서 받는" 행동은 커피에서 이미 대중화되었고, 50대 이용률도 50%에 달합니다. 이 검증된 행동을 **외식 예약으로 확장**하는 것이 잇테이블입니다.

**[경제적 변화 - E]** 외식업 소상공인 10명 중 6명이 노쇼 피해를 경험했고, 1회당 평균 손실액은 **44만 3천원**입니다(한국경제, 2026.01). 그런데 예약 보증금을 설정한 매장은 전체의 **14%에 불과**하고, 예약 방식의 **95.3%가 아직 전화**입니다. 외식업 종사자 수는 2024년 108만 명으로 전년 대비 1.6% 감소해 2021년 이후 처음으로 줄었습니다. 인건비 상승 + 인력 부족의 이중고 속에서 매장은 주문/결제 응대 부담을 줄여줄 디지털 솔루션이 절실합니다.

**[정치/제도적 변화 - P]** 2026년 스타트업 생태계 분위기 전망에서 긍정 42.5%, 정책 지원 강화 기대 34.1%로 창업 환경이 개선되고 있습니다. 또한 외식업 노쇼 사기 피해가 심각해지면서(2026년 상반기 2,892건, 414억 원 피해) 선결제/보증금 기반 예약에 대한 사회적 합의가 형성되고 있습니다.

**결론: 예약 앱은 "입장"만 예약하고, 테이블오더는 "착석 후"만 디지털화했습니다. "예약 시점에 메뉴+결제+조리 준비"까지 연결하는 서비스는 아직 없습니다. 이 빈 공간이 잇테이블의 기회입니다.**

## Q3. 아이디어는 누구의 어떤 문제를 해결해주나요? (필수, 1000자)

### [고객의 니즈 → 차별화 요소 → 경쟁우위]

**타겟 고객과 그들의 문제:**

| 고객 | 핵심 문제 | 기존 해결책의 한계 | 잇테이블의 해결 |
|------|---------|---------------|-----------|
| **직장인** (1순위) | 점심 1시간 중 실 식사시간 20분 | 캐치테이블: 자리만 예약, 착석 후 15~30분 추가 대기 | 메뉴+결제 사전 완료 → 도착 즉시 식사 시작 |
| **매장 점주** | 노쇼율 15~20%, 1회당 44만원 손실 | 전화 예약 95.3%, 보증금 설정 14%뿐 | 메뉴 전액 선결제 → 노쇼 구조적 차단 |
| **단체/회식** | 메뉴 합의 + 더치페이 정산 복잡 | 현장에서 10~15분 메뉴 고민, 결제 시 정산 분쟁 | 사전 메뉴 합의 + 분할 선결제 |

### [경쟁사 대비 차별화 (3C 분석)]

**Customers (시장):** 국내 외식 시장 110.8조 원(유로모니터, 2024). 이 중 예약/주문 디지털화 가능 시장(SAM) 약 12~15조 원. 초기 타겟(서울 내 샤브샤브/고깃집/전골) 시장(SOM) 약 1,500~3,000억 원.

**Competitors (경쟁사):**

| 비교 항목 | 캐치테이블 | 테이블오더(티오더) | **잇테이블** |
|---------|---------|-------------|----------|
| 핵심 가치 | 자리 예약 | 착석 후 주문 디지털화 | **자리+메뉴+결제+조리 준비 통합** |
| 고객 약속 | "예약 시간에 입장" | "직원 호출 없이 주문" | **"예약 시간에 즉시 식사 시작"** |
| 노쇼 대응 | 예약금 1~2만원 | 해당 없음 | **메뉴 전액 선결제** |
| 매장 효과 | 좌석 관리 | 주문 효율화 | **재료 예측+조리 타이밍+좌석+주문+결제 통합** |

> **캐치테이블이 "자리를 예약하는 서비스"라면, 잇테이블은 "먹을 메뉴와 식사 시작을 예약하는 서비스"입니다.**

**Company (자사 역량):**
- MVP 개발 완료: React Native 앱 + Node.js 백엔드 + 관리자/점주 대시보드
- 기존 자산: 화면 70+개, API 75+개, DB 테이블 32개, E2E 테스트 83건
- 핵심 기술: PortOne 선결제 연동, QR 체크인, Socket.IO 실시간 상태 알림, GPS ETA 계산
- 기존 밥동무(식사 모임 매칭) 서비스에서 검증된 보증금/신뢰도/체크인 시스템 재활용

### [경쟁우위 포지셔닝]

```
          예약 단계에서 해결하는 범위
          좌석만 ─────────────── 좌석+메뉴+결제+조리
            │                          │
캐치테이블  ●                          │
테이블링    ●                          │
            │                          │
잇테이블    │                          ● ← 유일한 포지션
            │                          │
```

### [수익 모델]

| 수익원 | 구조 | 비중 |
|--------|------|------|
| 거래 수수료 | 메뉴 결제금의 3~8% | 60% |
| 매장 SaaS | 월 29,000~79,000원 | 20% |
| 광고/프로모션 | 상위 노출, 배너 | 15% |
| 모임형 보증금 | 밥동무 기능 연결 | 5% |

**BEP 예상:** 월 GMV 약 10억 원(매장 200개+) 시 손익분기, Phase 2 중반(2027 Q1) 도달 목표.

### [6개월 내 검증 가설]

1. **고객이 메뉴까지 사전 확정+선결제를 선호하고 반복 이용하는가?** (전환율 60%+, 재예약률 30%+)
2. **매장이 도착 기반 조리 타이밍을 실제로 수행 가능한가?** (즉시 제공률 70%+)
3. **노쇼율이 기존 예약 대비 유의미하게 감소하는가?** (5% 이하, 업계 평균 15~20% 대비)

## 첨부 이미지 후보 (Q2, Q3 각 최대 5장)

### Q2 첨부 이미지 구성
1. **시장 성장 그래프** - 예약/웨이팅 앱 MAU 102만→291만 (185% 성장) 차트
2. **노쇼 피해 인포그래픽** - 10명 중 6명 피해, 1회 44만원, 보증금 설정 14%
3. **시장 공백 다이어그램** - 캐치테이블(자리) ↔ 테이블오더(착석후) ↔ 잇테이블(예약+주문+결제 통합)
4. **사용자 여정 비교** - 기존 플로우 vs 잇테이블 플로우 (대기시간 35분 → 0분)
5. **PEST 분석 요약도** - 정치/경제/사회/기술 4축 근거 자료 종합

### Q3 첨부 이미지 구성
1. **경쟁사 비교 포지셔닝 맵** - X축: 예약 범위, Y축: 디지털화 수준
2. **서비스 플로우** - 고객: 탐색→예약→메뉴→결제→도착→즉시 식사 / 점주: 접수→준비→조리→제공
3. **MVP 앱 스크린샷** - 실제 개발된 화면 (홈/매장상세/메뉴선택/결제/체크인)
4. **수익 모델 다이어그램** - 수수료 구조 + BEP 시뮬레이션
5. **TAM/SAM/SOM 파이 차트** - 110조→12~15조→1,500~3,000억
