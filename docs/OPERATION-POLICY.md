# 잇테이블 운영 정책서

> 최종 수정일: 2026-05-06 | 버전: v1.0

---

## 1. 예약 정책

### 1.1 예약 가능 조건
- 최소 예약 시간: 매장 운영시간 내, 현재 시각 기준 **최소 1시간 후**부터 예약 가능
- 최대 사전 예약: **30일 전**까지
- 최소 인원: **1명** / 최대 인원: **매장별 설정 (기본 상한 20명)**
- 예약 가능 시간 단위(슬롯): **30분 단위** (매장별 커스텀 가능: 15분/30분/60분)
- 동일 매장 동시 활성 예약: 고객 1인당 **최대 2건**
- 동일 시간대 타매장 예약: **허용** (중복 예약 경고 표시)

### 1.2 예약 확정 조건
- 결제 완료 시 즉시 확정 (자동)
- 결제 미완료 예약: **15분 내 미결제 시 자동 취소**
- 매장 수동 승인 모드: 매장 설정에 따라 결제 후에도 매장 확인 대기 가능 (최대 **30분 내 미응답 시 자동 확정**)

### 1.3 예약 변경
- 인원 변경: 예약 시간 **3시간 전**까지 가능 (추가 결제 또는 차액 환불 자동 처리)
- 시간 변경: 예약 시간 **3시간 전**까지, 해당 슬롯 가용 시에만 가능
- 메뉴 변경: 매장 조리 시작 전까지 가능 (조리 시작 후 변경 불가)
- 변경 횟수 제한: 예약 건당 **최대 3회**

## 2. 결제 정책

### 2.1 결제 수단
| 수단 | 지원 여부 | 비고 |
|------|----------|------|
| 신용/체크카드 | 지원 | PortOne PG 연동 |
| 카카오페이 | 지원 | 간편결제 |
| 네이버페이 | 지원 (Phase 2) | |
| 토스페이 | 지원 (Phase 2) | |
| 포인트 결제 | 지원 | 잇테이블 포인트, 전액 또는 혼합 결제 |
| 현금/계좌이체 | 미지원 | 선결제 모델 특성상 |

### 2.2 결제 금액 구성
```
총 결제금액 = 메뉴 합계 + 옵션 추가금 - 할인(쿠폰/프로모션) - 포인트 사용
```
- 최소 결제 금액: **1,000원**
- 최대 결제 금액: **5,000,000원** (1회)
- 부가세: 메뉴 가격에 **포함** (매장 등록 시 세금 포함가 입력)

### 2.3 분할 결제 (Phase 2)
- 모임형 예약 시 참가자별 메뉴 개별 결제 가능
- 호스트가 예약 생성 → 참가자 초대 → 각자 메뉴 선택 + 결제
- 전원 결제 완료 시 예약 확정 (미결제자 있을 경우 **예약 시간 2시간 전**까지 대기 후 호스트에게 알림)

## 3. 취소/환불 정책

### 3.1 고객 취소 (일반 예약)

| 취소 시점 | 환불 비율 | 수수료 | 비고 |
|----------|----------|--------|------|
| 예약 **24시간 전** 이전 | **100%** 전액 환불 | 없음 | |
| 예약 **3시간~24시간 전** | **90%** 환불 | 10% 위약금 | 매장 재료 준비 보상 |
| 예약 **1시간~3시간 전** | **70%** 환불 | 30% 위약금 | |
| 예약 **30분~1시간 전** | **50%** 환불 | 50% 위약금 | |
| 예약 **30분 이내** | **30%** 환불 | 70% 위약금 | 조리 시작 가능 구간 |
| **조리 시작 후** | **환불 불가** | 100% | 매장 '조리시작' 버튼 누른 후 |

> **소비자보호법 준수 참고:** 전자상거래법 제17조에 따라 서비스 제공 전까지는 청약 철회가 가능합니다. 조리 시작 전(예약 30분 이내)에도 최소 30% 환불을 보장하며, "조리 시작" 버튼은 매장이 실제로 식재료를 투입한 시점에만 누르도록 안내합니다. 고객에게는 결제 시 취소/환불 정책을 명확히 고지하고 동의를 받습니다.

### 3.2 환불 처리 기한
- 카드 결제: 취소 접수 후 **영업일 기준 3~5일** (카드사별 상이)
- 간편결제: 취소 접수 후 **즉시~1영업일**
- 포인트 결제: 취소 접수 후 **즉시** 포인트 복원
- 혼합 결제: 포인트 먼저 복원, 나머지 결제 수단으로 환불

### 3.3 매장 사유 취소
- 매장이 재료 소진, 영업 중단 등으로 취소할 경우:
  - **100% 전액 환불** (무조건)
  - 추가 **보상 쿠폰 3,000원** 지급
  - 매장 취소 누적 3회/월 시 **경고**, 5회/월 시 **임시 비활성화 (7일)**

### 3.4 시스템 장애 취소
- 결제 시스템 오류로 인한 예약 실패: 결제 자동 취소 + 전액 환불
- 결제 완료 후 시스템 장애: 고객 지원팀 접수 후 **24시간 내 처리**

## 4. 노쇼(No-show) 관리 정책

### 4.1 노쇼 판정 기준
| 조건 | 판정 |
|------|------|
| 예약 시간 **+20분** 경과 + 미도착 + 연락 불가 | **노쇼 확정** |
| 예약 시간 **+20분** 경과 + 고객 앱에서 "지연" 상태 | **추가 10분 대기** 후 미도착 시 노쇼 |
| 예약 시간 **+30분** 경과 (최종) | **무조건 노쇼 확정** (어떤 상태든) |
| 점주가 수동 노쇼 처리 | **즉시 노쇼 확정** (예약 시간 이후만 가능) |

### 4.2 노쇼 시 환불
- **원칙:** 환불 불가 (전액 매장 귀속)
- 단, 플랫폼 수수료는 노쇼 건에도 적용 (매장 정산 시 차감)

**예외 조항 (소비자보호법·약관법 준수):**
| 예외 사유 | 환불 비율 | 비고 |
|----------|----------|------|
| 매장이 조리를 시작하지 않은 경우 | **50%** 환불 | 매장 "조리시작" 버튼 미클릭 확인 |
| 불가항력 사유 (천재지변, 의료 긴급) | **100%** 환불 | 관리자 접수 후 증빙 확인, 48시간 내 처리 |
| 매장 귀책 (고객 연락 미응답, 좌석 미배정 등) | **100%** 환불 | 매장에 경고 처리 |
| 시스템 장애 (체크인 불가 등) | **100%** 환불 | 자동 감지 시 즉시 처리 |

> **근거:** 약관법 제6조(불공정 약관조항 금지) 및 소비자분쟁해결기준(공정위 고시)에 따라, 일률적 환불 불가는 불공정 약관으로 판단될 수 있습니다. 실제 손해(조리 시작 여부)를 기준으로 환불 범위를 차등 적용합니다.

### 4.3 노쇼 페널티 (고객)
| 누적 횟수 | 페널티 |
|----------|--------|
| 1회 | 경고 알림 + 신뢰도 하락 (밥알지수 -2.0) |
| 2회 (6개월 내) | 예약 시 **보증금 추가 10%** 부과 |
| 3회 (6개월 내) | **7일 예약 제한** |
| 5회 (12개월 내) | **30일 예약 제한** + 서비스 이용 경고 |
| 7회 (12개월 내) | **계정 정지 검토** (관리자 수동 판단) |

### 4.4 노쇼 페널티 해제
- 6개월간 노쇼 0건 시 누적 카운트 **1단계 감소**
- 정상 체크인 10회 연속 시 누적 카운트 **초기화**
- 불가항력 사유 (천재지변, 의료 긴급 등) 관리자 접수 시 **노쇼 취소 가능**

## 5. 도착 관리 정책

### 5.1 도착 예정 알림 체계
| 시점 | 고객 알림 | 매장 표시 |
|------|----------|----------|
| 예약 **2시간 전** | 푸시: "오늘 [매장명] 예약이 있습니다" | - |
| 예약 **1시간 전** | 푸시: "1시간 후 예약. 도착 예정을 알려주세요" | - |
| 예약 **30분 전** | 앱 내 도착 예정 버튼 활성화 | 고객 상태: "미확인" |
| 고객 '출발' 탭 | - | 고객 상태: "이동 중" + ETA 표시 |
| 매장 반경 500m | 자동 감지 알림 | 고객 상태: "매장 근처" |
| QR 체크인 | 체크인 완료 확인 | 고객 상태: "도착" → 조리 마무리 |

### 5.2 체크인 방식
- **QR 코드**: 매장 입구 QR 스캔 (Primary)
- **예약번호 입력**: 매장 태블릿에서 6자리 번호 입력 (Fallback)
- **GPS 자동 체크인**: 매장 반경 **50m** 진입 시 앱 내 체크인 버튼 활성화 (선택적)
- 체크인 유효 시간: 예약 시간 **-15분 ~ +30분**

### 5.3 지각 처리
| 상황 | 처리 |
|------|------|
| 예약 시간 +5분 | 매장에 "약간 지연" 표시 |
| 예약 시간 +10분 | 고객에 "서둘러 주세요" 푸시 |
| 예약 시간 +15분 | 매장에 "즉시 제공 보장 불가" 상태 전환 |
| 예약 시간 +20분 | 노쇼 판정 시작 (4.1 기준 적용) |

## 6. 조리 타이밍 정책

### 6.1 조리 상태 흐름
```
[대기] → [상차림 준비] → [조리 시작] → [조리 완료] → [제공 완료]
```

### 6.2 매장 권장 타이밍
| 고객 상태 | 권장 매장 액션 | 근거 |
|----------|--------------|------|
| 이동 중 (ETA 20분+) | 상차림 준비 (기본 세팅, 수저, 반찬) | 도착 전 테이블 준비 완료 |
| 이동 중 (ETA 10분 이내) | 조리 시작 (육수 데우기, 불판 예열 등) | 도착 즉시 음식 제공 |
| 매장 근처 / 체크인 | 조리 마무리 (음식 최종 플레이팅) | 착석 후 즉시 식사 |

### 6.3 "즉시 제공" 보장 정책

| 시간대 | 목표 제공 시간 | 보상 기준 |
|--------|--------------|----------|
| 일반 시간대 (오프피크) | 체크인 후 **5분 이내** | 10분 초과 시 포인트 500원 |
| 피크타임 (11:30~13:00, 18:00~19:30) | 체크인 후 **10분 이내** | 15분 초과 시 포인트 500원 |

- 매장 귀책 지연이 월 **5건 이상** 시 매장에 개선 요청 알림
- 피크타임 여부는 매장별 설정 가능 (기본값: 위 시간대)

### 6.4 매장 조리 시작 시점의 의미
- **조리 시작 버튼**을 누르면:
  - 환불 규칙이 "조리 시작 후"로 전환 (환불 불가)
  - 고객에게 "매장에서 조리를 시작했습니다" 푸시 알림
  - 해당 시점 기록 (분쟁 시 근거 자료)

## 7. 매장(점주) 정책

### 7.1 입점 조건
| 조건 | 요구사항 |
|------|---------|
| 사업자등록증 | 필수 (사업자번호 검증 API 연동) |
| 영업신고증 | 필수 |
| 통신판매업신고 | 불필요 (잇테이블이 통신판매중개자) |
| 계좌 정보 | 사업자 명의 계좌 (정산용) |
| 최소 메뉴 | 세트메뉴 **2개 이상** 등록 |
| 기기 요건 | 태블릿 또는 PC (점주 대시보드 접속용) |

### 7.2 수수료 구조
| 등급 | 조건 | 수수료율 | 비고 |
|------|------|---------|------|
| 기본 | 신규 매장 | **8%** | 메뉴 결제금의 8% |
| 볼륨 | 월 거래 500만원+ | **5%** | 자동 등급 전환 |
| 프랜차이즈 | 10개 매장+ 계약 | **3%** | 별도 계약 |
| 프리미엄 | 상위 노출 + 분석 | **5% + 월 49,000원** | SaaS 구독 포함 |

### 7.3 정산 주기
- 기본: **주 1회** (매주 월요일, 전주 월~일 정산)
- 빠른 정산: **D+3** (영업일 기준, 월 29,000원 추가)
- 정산 금액 = 고객 결제금 - 플랫폼 수수료 - 환불금 - PG 수수료
- 정산 최소 금액: **10,000원** (미달 시 다음 주로 이월)
- 세금계산서: 월 1회 자동 발행
- 정산 지연 시: **미정산 금액에 대해 연 이율 12% 지연이자 지급** (공정위 권고 준용)

> **정산 주기 로드맵:** MVP 단계에서는 주 1회 정산을 기본으로 하되, Phase 2에서 PG사 협의를 통해 **D+2~3 기본 정산**으로 전환을 검토합니다. 외식업 매장의 현금 흐름(재료비/인건비 주 단위 집행)을 고려한 결정입니다.

### 7.4 수수료 변경 정책
- 수수료율 변경 시: **최소 30일 전 서면(이메일/대시보드) 사전 통지**
- 매장 동의 거부 시: 기존 수수료율 유지 또는 퇴점 선택 가능
- 정산 내역/수수료 내역: 매장 대시보드에서 **실시간 확인 가능**
- 수수료 외 추가 비용(광고, 빠른 정산 등): 별도 동의 없이 자동 적용 불가

### 7.5 매장 비활성화/퇴점
| 사유 | 처리 |
|------|------|
| 매장 자발적 퇴점 | 진행 중 예약 완료 후 비활성화, 잔여 정산금 최종 지급 |
| 위생 문제 신고 3건+ | 임시 비활성화 + 관리자 확인 |
| 매장 취소 월 5회+ | 경고 → 임시 비활성화 (7일) |
| 고객 불만 누적 (별점 2.0 미만, 리뷰 20개+) | 개선 요청 → 미개선 시 퇴점 검토 |
| 사업자등록 말소 | 즉시 비활성화 |

## 8. 리뷰 정책

### 8.1 리뷰 작성 조건
- 체크인 완료 + 식사 완료 상태인 예약만 리뷰 작성 가능
- 예약 완료 후 **7일 이내** 작성 가능 (이후 작성 불가)
- 리뷰 작성 보상: 잇테이블 포인트 **200원**

### 8.2 리뷰 평가 항목

- 부적절 리뷰 신고: 욕설/광고/허위 → 관리자 검토 후 **24시간 내 처리**
- 매장 주 리뷰 답글: 무제한 (단, 욕설/비방 시 경고)

## 10. 모임형 예약 정책 (밥동무 연계)

### 10.1 모임 예약 프로세스
1. 호스트가 매장/시간/메뉴 선정 → 모임 생성
2. 참가자 모집 (최소 2명 ~ 최대 매장 수용 인원)
3. 각 참가자: 본인 메뉴 선택 + **메뉴 결제 + 참여 보증금**
4. 전원 결제 완료 → 예약 확정
5. 모집 마감 시간: 예약 **3시간 전** (미달 시 자동 취소 + 전액 환불)

### 10.2 보증금 규칙
| 항목 | 금액 | 비고 |
|------|------|------|
| 참여 보증금 | **3,000원** (고정) | 메뉴 결제와 별도 |
| 정상 출석 | 보증금 **100% 환급** (포인트 전환 선택 가능) | |
| 노쇼 (참가자) | 보증금 **환불 불가** + 잔여 참가자에게 보상 포인트 분배 | |
| 호스트 노쇼 | 보증금 환불 불가 + 전체 참가자 보증금 환급 + 보상 1,000P | |
| 모임 취소 (인원 미달) | **전액 환불** | 자동 처리 |

### 10.3 혼밥 메뉴 매칭
- 2인 이상 세트메뉴를 혼자 먹고 싶은 고객이 동행을 모집
- 매칭 성공 시: 각자 메뉴 선결제 → 일반 예약과 동일 프로세스
- 매칭 실패 시: 예약 3시간 전까지 미매칭 → 자동 취소 + 전액 환불

## 12. 개인정보 및 법률

### 12.1 수집 정보
| 항목 | 수집 목적 | 보유 기간 | 동의 방식 |
|------|----------|----------|----------|
| 이름, 전화번호 | 예약/본인 확인 | 탈퇴 후 30일 | 필수 동의 |
| 이메일 | 계정 관리, 알림 | 탈퇴 후 30일 | 필수 동의 |
| 결제 정보 | 결제/환불 처리 | 전자상거래법 5년 | 필수 동의 |
| 위치 정보 | 매장 검색, ETA 계산, 체크인 | 사용 즉시 폐기 | **별도 동의** (체크박스 분리) |
| 예약/주문 이력 | 서비스 제공, 추천 | 탈퇴 후 30일 (법적 보관 별도) | 필수 동의 |

> **위치정보법 준수:** 위치정보의 보호 및 이용 등에 관한 법률 제16조에 따라, 위치정보 수집·이용 동의는 개인정보 동의와 **별도 체크박스**로 분리하여 받습니다. GPS 기반 자동 체크인, ETA 전달 등 위치 정보가 매장에 전달되는 경우 **제3자 제공 동의**를 추가로 받습니다. 만 14세 미만 이용자의 경우 법정대리인 동의를 받습니다.

### 12.2 법적 지위
- 잇테이블: **통신판매중개자** (전자상거래법 제20조)
- 매장: **통신판매업자** (개별 사업자)
- 고객과 매장 간 직접 거래를 중개하는 플랫폼

### 12.3 분쟁 해결
1. 고객-매장 분쟁: 잇테이블 고객센터 중재 (접수 후 **48시간 내 1차 답변**)
2. 중재 불발 시: 한국소비자원 또는 관할 법원
3. 매장-플랫폼 분쟁: 서울중앙지방법원 관할

### 12.4 고객 불만 처리 절차
| 단계 | 시간 | 처리 내용 |
|------|------|----------|
| 1. 접수 | 즉시 | 앱 내 "문의하기" 또는 고객센터 전화 (운영시간 09:00~22:00) |
| 2. 1차 답변 | **24시간 내** | 담당자 배정 + 초기 답변 |
| 3. 조사/처리 | **48시간 내** | 매장 확인, 증거 검토, 환불/보상 결정 |
| 4. 결과 통보 | **72시간 내** | 최종 결과 고객 전달 (푸시 + 알림톡) |
| 5. 이의 신청 | 결과 통보 후 **7일 내** | 재검토 요청 가능 (1회) |

**고객 만족도 수집:** 분쟁 해결 후 24시간 내 만족도 설문 발송 (1~5점)

### 12.5 선결제 모델 관련 법적 고지 사항
- **결제 시 고지 필수 항목**: 취소/환불 정책, 조리 시작 시 환불 불가 기준, 노쇼 판정 기준
- **약관 동의**: 첫 예약 시 이용약관 + 취소/환불 정책 동의 (체크박스)
- **매장별 취소 정책 표시**: 예약 화면에 해당 매장의 취소 수수료 비율 명확히 표시
- **전자상거래법 준수**: 통신판매중개자로서 고객에게 매장 정보(사업자번호, 연락처) 제공 의무

### 12.6 통신판매중개자 면책 및 책임 범위

**면책 원칙 (전자상거래법 제20조 2항 준용):**
- 잇테이블은 고객과 매장 간 거래를 중개하며, 거래 당사자가 아닙니다
- 음식의 품질, 위생, 서비스에 대한 직접적 책임은 매장에 있습니다
- 단, 잇테이블의 고의·중과실(정보 오류, 시스템 장애, 허위 매장 방치 등)로 인한 손해는 잇테이블이 책임집니다

**매장 사업자정보 표시 의무:**
- 예약/결제 화면에 다음 정보 필수 표시:
  - 상호명, 대표자명, 사업자등록번호, 매장 주소, 연락처
- 매장 상세 페이지 하단에 전체 사업자정보 상시 노출

**위약금·보증금 산정 투명화:**
- 취소 위약금의 산정 근거(재료비, 조리 준비 비용 등)를 정책에 명시
- 노쇼 시 매장 귀속 금액의 근거(식재료 투입, 좌석 기회비용 등) 안내
- 공정위 소비자분쟁해결기준 준용 명시

### 12.7 6개월 내 검증 가설 (MVP 핵심 실험)

서비스 론칭 후 6개월 내 반드시 검증해야 할 3가지 가설:

| 가설 | 검증 지표 | 성공 기준 | 실패 시 대응 |
|------|----------|----------|------------|
| 1. 고객이 메뉴까지 사전 확정+선결제하는 경험을 선호하고 반복 이용하는가? | 선결제 전환율, 재예약률, 취소/변경 요청률 | 전환율 60%+, 재예약률 30%+, 취소율 15% 이하 | 부분 선결제(예약금만) 모드 도입 검토 |
| 2. 매장이 도착 기반 조리 타이밍 오퍼레이션을 실제로 수용/실행 가능한가? | 즉시 제공률, 매장 만족도, 오퍼레이션 오류 건수 | 즉시 제공률 70%+, 만족도 70%+ | 조리 타이밍을 필수→권장으로 전환, 매장 교육 강화 |
| 3. 노쇼/지각률이 기존 예약 대비 유의미하게 감소하는가? | 노쇼율, 지각률, 매장별 손실 감소 체감 | 노쇼율 5% 이하 (업계 평균 15~20% 대비) | 보증금 비율 상향, 노쇼 페널티 강화 |
